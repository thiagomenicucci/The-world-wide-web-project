You get and Empty document

The document sent back is empty, but there is no error message.

The inet daemon has started a process to run your server but it immediately failed. Possibilities include:

  • The daemon may not be in the file specified, or may not be executable by the specified user (or, if user id not specified in your variety of inetd.conf, root)
  • The rule file does not exist.
  • You have written your own daemomn and it crashes.
  • You are using ours and it crashes (mail us!)

Try running the daemon from a terminal window to see what happens.
